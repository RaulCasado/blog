El stack y qué hace cada pieza

Cada capa tiene un rol concreto en la generación o el servido.

🚀

Astro 7

Framework. Genera HTML estático, islas de JS solo donde hace falta. Orquesta rutas, colecciones y build.

📝

Content Collections + MDX

Los posts son .md en src/content/blog, validados por un esquema Zod (title, tags, pubDate…).

🎨

Shiki

Resalta los bloques de código en build. El HTML llega ya coloreado, sin JS de resaltado en cliente.

🌍

i18n propio

src/i18n/ui.ts (tabla de traducciones) + utils.ts (useTranslations). Rutas bajo [lang]/.

🔍

Fuse.js (lazy)

Búsqueda difusa en cliente, cargada bajo demanda contra un índice JSON compacto.

💬

Giscus

Comentarios vía GitHub Discussions, en un iframe aislado. Solo en las páginas de artículo.

📡

RSS + Sitemap

Feeds por idioma (@astrojs/rss) y sitemap automático para buscadores.

▲

Netlify (estático)

Sirve dist/ desde el edge. Procesa _redirects (301) y _headers (seguridad).

🖼️

astro:assets <Picture>

Optimiza imágenes en build: AVIF/WebP, srcset responsive. Clave del salto de rendimiento.

Flujo de datos en build

Lo que ocurre, en orden, cuando se ejecuta pnpm build.

1 · Carga de contenido

content.config.ts define la colección blog con un glob() sobre src/content/blog/** y un esquema Zod. Astro lee y valida cada .md.

2 · getStaticPaths()

Cada ruta dinámica declara qué URLs existen: recorre languages × (páginas / posts / tags / nº de página) y devuelve params + props.

3 · getLangPosts() — orden compartido

Helper en src/utils/posts.ts: filtra por idioma y ordena (featured primero, luego fecha). Lo usan índice, paginación, tags, RSS y el JSON de búsqueda → consistencia total.

4 · Render + Shiki + <Picture>

Cada página se renderiza a HTML. Shiki colorea el código; <Picture> genera derivados AVIF/WebP de las imágenes.

5 · dist/ + Netlify

Sale HTML estático + el JSON de búsqueda, RSS, sitemap, _redirects y _headers. Netlify lo sirve desde el edge.

Cómo se carga cada cosa en el navegador

Qué llega de entrada, qué es diferido y qué está aislado.

Inmediato
  • •HTML estático ya renderizado (incluido el código coloreado por Shiki).
  • •Imágenes vía <Picture> con srcset; el navegador elige el tamaño.
  • •CSS scoped por componente, mínimo.
Diferido (lazy)
  • •Fuse.js + JSON de búsqueda: solo al enfocar el buscador (import() dinámico).
  • •Imágenes no-hero: loading="lazy".
  • •Carrusel de tags: script ligero al cargar el índice.
Aislado / externo
  • •Giscus: iframe de giscus.app, solo en artículos. Aislado por same-origin.
  • •Fuentes: preload de Atkinson woff local (sin CDN externa).
